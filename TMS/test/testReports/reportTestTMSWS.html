Tests methodology

The WS tests are made using JUnit framework. They are based on a subset of the test scenarios described in the UMS test plan (reference D2.1b). The objective of these tests is to confirm that the service is executed without exception in a normal case, and to confirm the right exception is raised in an exception case.

The tests qualified as "simple" on a given functionality of the UMS module correspond to a functional test of the main scenario without optional parameters. The tests of the optional parameters have been already done for the C++ API on which this WS interface is directly based.
